## What happened

EMIS built full multi-provider LLM support. CPDS-AI inherits this architecture by design.

## What CPDS-AI needs to know about provider switching

When CPDS-AI builds its own LLM calls (curation helper, future analysis), it uses the
same pattern established in EMIS. Non-negotiable architecture rule:

- CLI flag: `--provider anthropic|ollama|groq|cerebras|openrouter|custom`
- Env var fallback: `LLM_PROVIDER=cerebras` in `.env`
- Model names: always from env var, never hardcoded
- No code changes required to switch providers
