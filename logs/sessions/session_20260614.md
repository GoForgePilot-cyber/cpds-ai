## What was decided this session

### Strategic decisions (locked)
- Platform: Beehiiv Scale ($43/mo) — not Substack. Reasons: 0% revenue cut, native ad network, paywall section gating, Beehiiv API for automation, better multi-stream monetization.
- Free/paid split architecture: free section = descriptive (what's happening), paid section = prescriptive (what to do). Paywall break mid-post via Beehiiv section visibility gating.
- Pricing: $19/month, $190/year (annual as default to reduce churn).
- Source stack: 3-tier architecture. Tier 1 (free APIs) immediate; Tier 2 (commercial signals) post-vertical-selection; Tier 3 (Discord bot, Exploding Topics) post-launch.
- Recursive self-improvement: self_improve.py runs at every phase milestone and on source health degradation. milestone_analysis.py runs deeper business review at phase transitions.

### Architecture decisions (locked)
- curate.py makes TWO separate Claude API calls: free section (descriptive) and paid section (prescriptive). This separation is intentional — different objective, different prompt, different output.
- Always push to Beehiiv as `draft` status — never auto-send. Human reviews before every send.
- Social posts generated by social_post.py, saved as draft, human edits before posting.
- Community posts (Reddit/Discord) never automated — human writes from social_post.py opener.

### Vertical selection (NOT decided)
- Waiting for ~2026-06-28 EMIS 2-week signal run
- Candidates ranked: (1) AI billing infra [91/100 EMIS score, sparse newsletter competition], (2) n8n automation services [strong signal, price-sensitive audience], (3) niche microsaas [incumbent Micro SaaS Idea has 40K+ subs]
- AI billing infra is recommended but EMIS data decides
